Yep—this is a classic “desktop overlay + center text” layout leaking into mobile. Two things cause the overlap:

Absolute-positioned chips/pills sitting on top of content on small screens.

Containers inheriting text-center and justify-center, so everything centers and collides.

Below is a mobile-first EventCard you can drop in. It stacks cleanly on phones (no overlays), and only overlays the category pill on md+. All meta rows wrap instead of overlapping.

import { MapPin, Calendar as CalendarIcon, Users } from "lucide-react";
import { Button } from "@/components/ui/button";

type Event = {
  id: number;
  title: string;
  description?: string;
  imageUrl?: string;
  category?: string;
  address?: string;       // "3200 The Strand, Manhattan Beach, CA"
  date?: string;          // ISO or already formatted
  attendingCount?: number;
};

export function EventCard({ event }: { event: Event }) {
  return (
    <div className="rounded-2xl bg-gray-800/40 dark:bg-gray-800 shadow-sm overflow-hidden">
      {/* Image + category */}
      <div className="relative">
        <img
          src={event.imageUrl}
          alt={event.title}
          className="w-full aspect-[16/9] object-cover"
          loading="lazy"
        />
        {/* On mobile this renders inline below; on md+ it overlays */}
        <div className="hidden md:block">
          {event.category && (
            <span className="md:absolute md:top-3 md:right-3 inline-block rounded-full px-3 py-1 text-xs font-semibold bg-gray-900/80 text-white">
              {event.category}
            </span>
          )}
        </div>
      </div>

      {/* Content */}
      <div className="p-4 md:p-5 text-left">
        {/* Mobile category (not overlay) */}
        <div className="md:hidden mb-2">
          {event.category && (
            <span className="inline-block rounded-full px-3 py-1 text-xs font-semibold bg-gray-900/80 text-white">
              {event.category}
            </span>
          )}
        </div>

        <h3 className="text-lg font-semibold text-white leading-snug line-clamp-2">
          {event.title}
        </h3>

        {event.description && (
          <p className="mt-1 text-sm text-gray-300 leading-relaxed line-clamp-3">
            {event.description}
          </p>
        )}

        {/* Meta row(s) — wrap on small screens to avoid overlap */}
        <div className="mt-3 flex flex-col gap-2 sm:flex-row sm:flex-wrap sm:items-center">
          {event.address && (
            <div className="min-w-0 flex items-center gap-2 text-sm text-gray-300">
              <MapPin className="h-4 w-4 shrink-0" />
              <span className="truncate">{event.address}</span>
            </div>
          )}
          {event.date && (
            <div className="min-w-0 flex items-center gap-2 text-sm text-gray-300">
              <CalendarIcon className="h-4 w-4 shrink-0" />
              <span className="truncate">{event.date}</span>
            </div>
          )}
          <div className="min-w-0 flex items-center gap-2 text-sm text-gray-300">
            <Users className="h-4 w-4 shrink-0" />
            <span className="truncate">
              {event.attendingCount ?? 0} attending
            </span>
          </div>
        </div>

        {/* Actions */}
        <div className="mt-4 flex gap-2 sm:items-center sm:justify-between">
          <div className="flex gap-2 flex-1 sm:flex-none">
            <Button className="flex-1 sm:flex-none">Join</Button>
            <Button variant="outline" className="flex-1 sm:flex-none">
              More
            </Button>
          </div>
        </div>
      </div>
    </div>
  );
}

Why this fixes your screenshot

No absolute chips on mobile. Overlays are hidden on small screens and moved into normal flow → nothing sits on top of other text.

Text is left-aligned (text-left), with min-w-0, truncate, and flex-wrap on meta rows so long addresses/dates wrap gracefully.

Aspect-ratio image (aspect-[16/9]) prevents jumpiness and ensures the content below doesn’t slide under the image.

If you want to keep the attendance/category pills over the image on mobile

Keep them absolute but reserve space so they won’t collide with content below:

<div className="relative pb-8">  {/* <-- adds bottom space for pills */}
  <img className="w-full aspect-[16/9] object-cover" ... />
  <div className="absolute bottom-2 left-2 right-2 flex flex-wrap gap-2">
    <!-- your pills -->
  </div>
</div>


…but the stacked layout above is much more reliable on phones.
