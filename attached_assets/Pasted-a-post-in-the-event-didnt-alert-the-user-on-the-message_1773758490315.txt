Good question — here's the right approach:

Yes, notify — but smartly. Don't notify every single message or it becomes spam.

Rules:

First message after you've been away — notify once, then batch the rest. "3 new messages in Sing Along"
If you're actively in the chat — no notification
@mention — always notify immediately
New person joins the chat — notify existing members once
Don't notify your own messages — obviously
For activity feed:

Event chat messages should NOT spam the activity feed — that's for social actions (connections, vouches, RSVPs). Keep messages in the notification bell only, not activity feed.
