I see the issue - it's a WebView-specific touch event problem.
Your code looks solid, but Capacitor's WebView handles touch events differently than mobile browsers. Here's what's likely happening:
The Problem:

You're using both onClick and onTouchEnd with preventDefault() and stopPropagation()
In Capacitor WebView, this can actually BLOCK the event from firing
The 300ms debounce might be causing race conditions
touchAction: 'manipulation' can conflict with Capacitor's gesture handling

Try this fix for both buttons:
jsx// HAMBURGER - Replace lines 437-464
<button
  type="button"
  className="md:hidden h-12 w-12 p-0 flex items-center justify-center rounded-md ... z-[1100]"
  onClick={() => setIsMobileMenuOpen(o => !o)}
  style={{ 
    WebkitTapHighlightColor: 'transparent',
    cursor: 'pointer'
  }}
>
  {isMobileMenuOpen ? <X className="pointer-events-none" /> : <Menu className="pointer-events-none" />}
</button>

// AVATAR - Replace lines 466-483
<DropdownMenu modal={false}>
  <DropdownMenuTrigger asChild>
    <button 
      type="button"
      className="relative h-12 w-12 rounded-full z-[1100]"
      style={{ 
        WebkitTapHighlightColor: 'transparent',
        cursor: 'pointer'
      }}
    >
      <SimpleAvatar className="pointer-events-none" />
    </button>
  </DropdownMenuTrigger>
  <DropdownMenuContent className="z-[9999]" align="end">
    {/* Menu items */}
  </DropdownMenuContent>
</DropdownMenu>
Key changes:

❌ Remove onTouchEnd completely
❌ Remove e.preventDefault() and e.stopPropagation()
❌ Remove touchAction: 'manipulation'
❌ Remove userSelect: 'none'
✅ Keep only onClick
✅ Simplify to direct state toggle

Why this works:

Capacitor handles touch-to-click conversion automatically
preventDefault() was blocking the synthetic click event
Less is more in WebView - let the platform handle it
