Totally get it—let’s isolate the mobile changes so desktop stays exactly how you designed it. Below are drop-in fixes that (1) scope mobile tweaks to < md, (2) restore desktop layouts, and (3) keep pills and hero sane across breakpoints.

1) Navbar: menu visible only on mobile, never on desktop

Keep your desktop nav as-is (hidden md:flex).

Ensure the mobile menu overlay is mobile-only with md:hidden and sits above the header.

{/* Mobile menu overlay (mobile-only) */}
<div
  id="mobile-menu"
  className={`md:hidden fixed inset-x-0 top-20 z-[100] bg-white dark:bg-black border-b border-gray-200 dark:border-gray-800 shadow-lg transition-[max-height,opacity] duration-200 overflow-hidden ${isMobileMenuOpen ? "max-h-[85svh] opacity-100" : "max-h-0 opacity-0 pointer-events-none"}`}
>
  {/* …menu content… */}
</div>


If you used the portal version, keep it—but make the menu root carry md:hidden (so at desktop sizes it’s never rendered/visible even with z-index):

const menu = (
  <div id="mobile-menu" className={`md:hidden fixed inset-x-0 top-20 z-[100] …`}>
    {/* … */}
  </div>
);

3) Pills: same look, responsive sizing so desktop spacing isn’t wrecked

Use a single class string that scales with breakpoints:

const pill = "inline-flex items-center justify-center rounded-full font-medium whitespace-nowrap leading-none bg-blue-500 text-white border-0 h-7 md:h-8 px-3 text-[11px] md:text-xs";


Usage:

<Badge className={pill}>Beach</Badge>
<span className={pill}>Los Angeles</span>


Wrapper must allow wrapping:

<div className="flex flex-wrap gap-2">
  {/* pills */}
</div>


If you have a “+N” chip:

<Badge className="inline-flex items-center justify-center rounded-full h-7 md:h-8 px-2 text-[11px] md:text-xs leading-none border border-gray-300 dark:border-gray-600 text-gray-600 dark:text-gray-300 bg-transparent">
  +{extra}
</Badge>

4) Hero height: taller on phones, moderate on desktop

If the hero got huge on desktop, make it mobile-first and reduce at larger breakpoints:

<section className="min-h-[68svh] md:min-h-[56vh] xl:min-h-[48vh] bg-cover bg-center bg-no-repeat">
  {/* hero content */}
</section>


Also change any forced single-line headings:

className="whitespace-normal sm:whitespace-nowrap"

5) Map overflow: don’t clip on desktop

We likely added overflow-hidden globally. Scope it to mobile and relax at md+:

<div className="rounded-2xl border max-w-full overflow-hidden md:overflow-visible">
  <CityMap ... />
</div>


Popup wrapping (safe for all sizes; keep this in global CSS):

.mapboxgl-popup-content, .gm-style .gm-style-iw, .leaflet-popup-content {
  max-width: calc(100vw - 2rem);
  overflow-wrap: anywhere;
  word-break: break-word;
  white-space: normal;
}

6) One global rule to revert (it can mess with desktop)

If you added this earlier, remove it from globals—it can shrink desktop layouts unexpectedly:

/* REMOVE this global rule if present */
* { min-width: 0; }


Instead, add min-w-0 only where needed (flex children that contain long text):

<div className="flex gap-3">
  <div className="min-w-0"> {/* text content that might truncate */} … </div>
  <div>…</div>
</div>

Quick test checklist

At phone width (<768px): mobile menu opens; connections show grid; pills don’t overlap; hero fills ~⅔ screen; no horizontal scroll.

At desktop (≥1024px): original desktop nav & connections layout; hero is shorter; map panel doesn’t clip.
