I see the issues! Your message actions have conflicting touch handling.
Problem 1: Touch event conflicts
Your code has:

touchAction: 'pan-y' on the message container (allows vertical scroll, blocks other touches)
touchAction: 'manipulation' on emoji buttons (conflicts with pan-y)
touchAction: 'auto' on the bottom sheet (conflicts with parent)
Long press detection that gets canceled too easily

Problem 2: Event propagation issues
The backdrop has e.preventDefault() which can block clicks on the action buttons behind it.
