If you’re still seeing non-black text even though you’ve got text-black, that means something else is overriding it (most commonly: a Button/variant default like text-white / text-primary-foreground, or a child element/icon with its own text-* class).
Fast, guaranteed fix (force override)
Change text-black → !text-black (and do the same for hover/focus if needed):
“View Travel Plans / View Upcoming Trips”
className="bg-gradient-to-r from-orange-600 to-red-500 hover:from-orange-700 hover:to-red-600 !text-black border-0 px-4 py-2 text-sm rounded-lg shadow-md transition-all"

If you have icons inside the buttons
Sometimes the text is black but the icon is inheriting a different color. Force it too:
<SomeIcon className="!text-black" />

If you’re using a shared <Button> component (shadcn/ui etc.)
Those variants often include text-white / text-primary-foreground. !text-black fixes it immediately.
If you want it “cleaner” long-term, you can also remove the default text color from the variant you’re using (but the !text-black route is the quickest and most reliable).
If you paste one actual button JSX (the full <Button ...>...</Button>), I can point to exactly what’s overriding it (usually it’s in the base Button variant styles or a child span).
