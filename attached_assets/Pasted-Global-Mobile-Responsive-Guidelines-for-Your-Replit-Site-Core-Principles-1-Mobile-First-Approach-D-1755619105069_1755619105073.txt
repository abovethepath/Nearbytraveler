Global Mobile Responsive Guidelines for Your Replit Site
Core Principles
1. Mobile-First Approach

Design for mobile screens first (320px - 768px)
Use Tailwind's responsive prefixes: sm:, md:, lg:, xl:
Default styles apply to mobile, larger screens get overrides

2. Responsive Breakpoints
css- Mobile: 0-639px (default)
- sm: 640px+ (tablets)
- md: 768px+ (small laptops)
- lg: 1024px+ (desktops)
- xl: 1280px+ (large screens)
Key Implementation Patterns
Text Sizing
tsx// Headings
<h1 className="text-2xl sm:text-3xl md:text-4xl lg:text-5xl">

// Body text
<p className="text-sm sm:text-base md:text-lg">

// Small text
<span className="text-xs sm:text-sm">
Spacing & Padding
tsx// Containers
<div className="px-4 sm:px-6 lg:px-8">

// Vertical spacing
<div className="py-4 sm:py-6 lg:py-8">

// Margins
<div className="mb-4 sm:mb-6 lg:mb-8">
Grid Layouts
tsx// Responsive grid
<div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-4 sm:gap-6">

// Flex layouts
<div className="flex flex-col sm:flex-row gap-4">
Buttons
tsx// Full width on mobile, auto on larger screens
<Button className="w-full sm:w-auto">

// Responsive text and padding
<Button className="text-xs sm:text-sm px-3 sm:px-4 py-1.5 sm:py-2">
Cards & Modals
tsx// Card responsive padding
<Card className="p-4 sm:p-6">

// Modal with max width
<div className="w-full max-w-sm sm:max-w-md lg:max-w-lg">
Component-Specific Updates
Navigation

Stack navigation items vertically on mobile
Use hamburger menu for mobile navigation
Hide less important items on small screens

Forms

Stack form fields vertically on mobile
Full-width inputs on mobile
Larger touch targets (min 44px height)

Images

Use responsive image sizing
Consider lazy loading for performance
Provide mobile-optimized images when possible

Tables

Make tables horizontally scrollable on mobile
Consider card-based layouts for mobile instead of tables
Hide non-essential columns on small screens

Performance Optimizations
1. Viewport Meta Tag
Ensure your HTML includes:
html<meta name="viewport" content="width=device-width, initial-scale=1.0">
2. Touch-Friendly Interactions

Minimum touch target size: 44x44px
Add proper spacing between clickable elements
Use hover: states only with sm: prefix for desktop

3. Loading States

Show skeleton loaders on mobile
Optimize initial load for mobile networks
Lazy load heavy components

Testing Checklist

 Test on actual mobile devices (iOS & Android)
 Check all breakpoints (320px, 375px, 414px, 768px, 1024px)
 Verify touch interactions work properly
 Ensure text is readable without zooming
 Check that modals and overlays work on mobile
 Verify forms are usable on mobile keyboards
 Test landscape orientation on mobile
 Check performance on 3G/4G connections

Common Pitfalls to Avoid

Fixed widths - Use max-width instead
Small touch targets - Keep clickable areas large
Horizontal scrolling - Use overflow-x-auto when needed
Dense layouts - Add breathing room on mobile
Desktop-only hover states - Provide alternative interactions

Implementation Priority

High Priority

Navigation responsiveness
Core content readability
Primary CTAs accessibility
Form usability


Medium Priority

Image optimization
Card layouts
Secondary features


Low Priority

Decorative elements
Advanced animations
Nice-to-have features



Quick Wins for Your Site

Add w-full to all buttons on mobile, then sm:w-auto for larger screens
Use text-sm as default, then sm:text-base for larger screens
Replace fixed padding with responsive padding (p-4 sm:p-6)
Stack all multi-column layouts on mobile (flex-col sm:flex-row)
Ensure all modals have proper mobile viewport constraints

CSS Variables for Consistency
Consider adding these to your global CSS:
css:root {
  --mobile-padding: 1rem;
  --tablet-padding: 1.5rem;
  --desktop-padding: 2rem;
  --mobile-text: 0.875rem;
  --desktop-text: 1rem;
}
Apply these patterns consistently across all your pages for a cohesive mobile experience!
