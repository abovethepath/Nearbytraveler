— here are surgical edits to make your Home page behave on phones: smaller/safer hero, no horizontal scroll, and a modal that fits the viewport. These changes target home.tsx. 

1) Fix the hero section height + background

Replace the <section> opening tag (remove the inline minHeight: '500px'):

<section
  className="relative overflow-hidden text-white min-h-[60svh] sm:min-h-[70svh] md:min-h-[72svh] bg-cover bg-center bg-no-repeat"
  style={{
    backgroundImage: "url('/attached_assets/beach travel_1754973619241.jpg')",
  }}
>


Why: svh tracks the safe mobile viewport, so the hero isn’t too tall or jumpy on iOS.

2) Prevent heading overflow on narrow phones

Your headings use whitespace-nowrap, which can force overflow on small screens. Change the two places with className="whitespace-nowrap text-center" to:

className="whitespace-normal sm:whitespace-nowrap text-center"


This keeps single-line look on tablets/desktop but allows wrapping on phones.

3) Make the Advanced Filters modal fit the screen

Change the overlay padding slightly and constrain the Card width/height so it never overflows:

{/* Overlay */}
<div className="fixed inset-0 bg-black/50 z-50 flex items-center justify-center p-3 sm:p-4">

  {/* Outer card */}
  <Card className="w-[min(100vw-1.5rem,56rem)] max-h-[85svh] overflow-y-auto p-4 sm:p-6 rounded-xl border-2 border-gray-200 dark:border-gray-700 shadow-lg bg-white dark:bg-gray-800">
    {/* Inner card */}
    <Card className="p-4 sm:p-6 mx-0 sm:mx-0 rounded-xl border-2 border-gray-200 dark:border-gray-700 shadow-lg">
      ...
    </Card>
  </Card>
</div>


Notes:

w-[min(100vw-1.5rem,56rem)] keeps it inside the viewport with a small gutter.

max-h-[85svh] + overflow-y-auto guarantees it scrolls instead of spilling off-screen.

4) Global “no horizontal scroll” + text flex fixes

Add these once to your global CSS (e.g., src/index.css or globals.css):

/* Guard rails for mobile */
html, body, #root { width: 100%; overflow-x: hidden; }
* { min-width: 0; } /* prevents long text from blowing up flex children */

5) (Optional) Buttons & pills that never wrap weirdly

If you see misaligned pill rows, add whitespace-nowrap leading-none on the pill class you reuse:

className="inline-flex items-center justify-center h-8 rounded-full px-3 text-sm font-medium whitespace-nowrap leading-none bg-blue-500 text-white border-0"
