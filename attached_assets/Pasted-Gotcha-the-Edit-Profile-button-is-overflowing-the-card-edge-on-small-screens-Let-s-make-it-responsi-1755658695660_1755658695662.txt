Gotcha—the Edit Profile button is overflowing the card edge on small screens. Let’s make it responsive so it never sticks out:

Drop-in fix for your About card header

Replace your <CardHeader> block with this:

<CardHeader className="flex items-center gap-2 flex-wrap min-w-0 w-full">
  <CardTitle className="text-base sm:text-lg truncate max-w-[calc(100%-3rem)] sm:max-w-none">
    About {user?.userType === 'business'
      ? (user?.businessName || user?.name || user?.username)
      : (user?.username || 'User')}
  </CardTitle>

  {isOwnProfile && (
    <>
      {/* Icon-only on phones */}
      <Button
        size="icon"
        variant="outline"
        onClick={() => setIsEditMode(true)}
        className="ml-auto sm:hidden shrink-0"
        aria-label="Edit Profile"
      >
        <svg viewBox="0 0 24 24" className="w-4 h-4" fill="currentColor">
          <path d="M3 17.25V21h3.75L17.81 9.94l-3.75-3.75L3 17.25zm2.92 2.33H5v-.92l8.06-8.06.92.92L5.92 19.58zM20.71 7.04a1 1 0 0 0 0-1.41l-2.34-2.34a1 1 0 0 0-1.41 0l-1.83 1.83 3.75 3.75 1.83-1.83z"/>
        </svg>
      </Button>

      {/* Labeled button on ≥ sm */}
      <Button
        size="sm"
        variant="outline"
        onClick={() => setIsEditMode(true)}
        className="ml-auto hidden sm:inline-flex shrink-0 whitespace-nowrap"
      >
        Edit Profile
      </Button>
    </>
  )}
</CardHeader>

Keep content from clipping

Make sure your <Card> and <CardContent> allow wrapping and don’t create inner scrollbars:

<Card className="mt-6 relative overflow-visible">
  <CardContent className="space-y-4 pr-3 sm:pr-4 min-w-0 break-words overflow-visible">
    {/* ... */}
  </CardContent>
</Card>

Remove any old overlay Edit button

Search and delete any button like:

<Button className="absolute right-2 bottom-2 ...">Edit Profile</Button>


Those overlays are what tend to hang off the edge.

(Optional) tiny CSS helper (if you saw mini scrollbars)

Add once to your global CSS:

.no-scrollbar { -ms-overflow-style:none; scrollbar-width:none; }
.no-scrollbar::-webkit-scrollbar { display:none; }


Then add no-scrollbar to any container that shouldn’t scroll.

This makes the edit control icon-only on phones (fits inside) and a full label on tablet/desktop, so it won’t spill off the card in narrow Replit preview widths.
