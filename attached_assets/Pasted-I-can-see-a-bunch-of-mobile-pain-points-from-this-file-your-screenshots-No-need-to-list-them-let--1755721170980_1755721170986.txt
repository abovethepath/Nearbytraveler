I can see a bunch of mobile pain points from this file + your screenshots. No need to list them—let’s land fixes you can copy-paste right now. These are safe, targeted tweaks that won’t blow up desktop.

0) One tiny utility in your global CSS (once)

In src/index.css (or your Tailwind globals), add:

/* mobile-rescue */
@layer utilities {
  .no-scrollbar::-webkit-scrollbar { display: none; }
  .no-scrollbar { -ms-overflow-style: none; scrollbar-width: none; }
}


This gives us a no-scrollbar helper we can reuse.

1) Stop horizontal wobble app-wide

Your profile root needs to hide x-overflow so micro chips/rows don’t force a horizontal scrollbar on small screens.

Find:

Replace with:

<div className="min-h-screen bg-gray-50 dark:bg-gray-900 overflow-x-hidden">

Why this fixes what you’re seeing

Pill scrollbars / micro arrows → no-scrollbar removes them in WebKit/Firefox; whitespace-nowrap prevents mid-chip wrapping that causes tiny vertical overflow.

Tabs overflow → 2 columns on xs avoids squeezing 4 triggers into one row.

Popover clipping → sets mobile-friendly width and height; no scrollbars inside.

Dialogs → constrained to 92vw on mobile; no off-screen content.

Text overflow → break-words, truncate, and min-w-0 on flex children stop long strings from pushing the layout.

Horizontal page scroll → overflow-x-hidden at the page root.

If you paste these small changes and hard-refresh, your mobile layout should stop overflowing, chips stop showing scrollbars, popovers and modals fit the screen, and the tabs won’t wrap awkwardly.
