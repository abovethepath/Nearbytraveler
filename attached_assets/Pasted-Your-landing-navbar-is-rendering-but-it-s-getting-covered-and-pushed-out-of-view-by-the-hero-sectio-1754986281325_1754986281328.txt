Your landing navbar is rendering, but it’s getting covered and pushed out of view by the hero section’s absolutely-positioned overlay + stacked paddings. Also, the page isn’t using a flex column layout, so the footer + section paddings create that “mile” of empty space.

Here’s a tight, drop-in fix that (1) guarantees the landing navbar shows, (2) removes the extra bottom space, and (3) keeps the mobile menu usable.

What to change in the landing page
1) Pin the landing navbar to the very top, above everything
Wrap it in a fixed container and add a spacer so the content doesn’t sit under it.

tsx
Copy
Edit
{/* NAVBAR (fixed, always visible) */}
<div className="fixed inset-x-0 top-0 z-[100]">
  <LandingNavbar />
</div>

{/* Spacer equal to navbar height so content starts below it */}
<div className="h-16 md:h-[72px]" />
This beats any stacking-context/overlay quirks. Your LandingNavbar component uses relative z-50 — that’s fine; the wrapper is what’s fixed/z-indexed now.

2) Make the page a proper flex column & dock the footer
Use dvh (mobile safe viewport height), and let the footer sit flush without extra padding.

tsx
Copy
Edit
// Top-level wrapper
<div className="min-h-dvh flex flex-col bg-gray-50 dark:bg-gray-900 overflow-x-hidden">

  {/* ... navbar (fixed) and spacer go here ... */}

  {/* Main content takes remaining space */}
  <main className="flex-1">
    {/* hero + sections */}
  </main>

  {/* Footer pinned to bottom */}
  <footer className="mt-auto">
    <Footer />
  </footer>
</div>
3) Make the hero overlay non-interactive and trim paddings
The overlay should never block taps on the navbar or hamburger.

4) Kill hidden bottom padding
Remove any pb-20, pb-32, mb-20 from sections near the bottom.

In your Footer component, keep padding small, e.g. py-6 md:py-8, and avoid extra mb-*/pb-*.

If the mobile menu still won’t open
Your LandingNavbar looks solid. With the navbar wrapped in fixed and the hero overlay set to pointer-events-none, the hamburger should work. If it doesn’t:

Ensure no ancestor of the navbar has overflow-hidden and is position: sticky (we’re using a fixed wrapper now, so this shouldn’t matter).

If the mobile menu should overlay content, add z-[110] to the menu container:

Why this fixes your screenshot
The huge empty area was a combination of stacked paddings and not using flex to anchor the footer. min-h-dvh flex flex-col with mt-auto on the footer removes that extra space.

Fixing the navbar in a fixed wrapper with z-[100] guarantees it sits above the hero overlay and is always visible/tappable.
