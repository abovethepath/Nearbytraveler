paste into your React/Tailwind project on Replit. These fix the most common mobile problems (overflow, cramped grids, weird 100vh on iOS, misaligned pills, etc.).

1) Global mobile fixes (add once)

src/index.css or globals.css

/* --- Mobile sanity fixes --- */
:root { color-scheme: light; }

/* iOS 100vh fix */
.min-screen { min-height: 100dvh; }

/* Prevent random horizontal scroll on small screens */
html, body, #root { width: 100%; overflow-x: hidden; }

/* Images and media scale down nicely */
img, video, canvas, svg { max-width: 100%; height: auto; }

/* Tailwind + flex children can overflow; this prevents text pushing layout */
* { min-width: 0; }

/* iOS Safari tap highlight */
* { -webkit-tap-highlight-color: rgba(0,0,0,0.1); }

/* Prevent zoom-on-focus on iOS (keep at least 16px font size on inputs) */
input, select, textarea, button { font-size: 16px; }

/* Reusable container */
.container-default {
  @apply max-w-screen-xl mx-auto px-4 sm:px-6 lg:px-8;
}

/* Optional: your pill class from earlier */
.pill {
  @apply inline-flex items-center justify-center h-8 rounded-full px-3 text-sm font-medium whitespace-nowrap leading-none bg-blue-500 text-white border-0;
}


Make sure your HTML has the viewport meta tag (usually already there):

<meta name="viewport" content="width=device-width, initial-scale=1" />

Why this stays clean on phones

Grid templates switch from 1-column → 2/3-column at md/lg.

flex-wrap gap-2 on the pills prevents overflow.

aspect-[4/3] keeps images from stretching and avoids layout shift.

min-screen (100dvh) keeps pages tall enough on iOS without jumpiness.

6) If something still breaks on mobile

A single element with w-[800px]/min-w-* or absolute can cause full-page overflow. Add max-w-full or remove fixed widths.

Put overflow-x-hidden on a section, not the whole page, if a specific child is misbehaving.

Add min-w-0 to every flex child that contains text or chips (this prevents text from forcing the container wider).

For icons in pills: add gap-1.5 [&>svg]:h-4 [&>svg]:w-4.
