Here's a complete breakdown of your signup/registration system. It's a 3-step flow:

STEP 1: Choose Account Type (/join or /signup)

User picks one of three types:
Nearby Local (type: "local") - "I live here & want to meet travelers"
Nearby Traveler (type: "traveler") - "I'm traveling & want to connect"
Nearby Business (type: "business") - "I run a local business"
Selection is stored in sessionStorage as selectedUserType
Then navigates to /signup/account
STEP 2: Create Account (/signup/account)

Collects these fields (all required):
Full Name (or "Business Name" if business type)
Username (6-12 characters, real-time availability check via GET /api/check-username/{username})
Email
Confirm Email
Phone Number (US format auto-formatting)
Password (min 8 characters, with show/hide toggle)
Confirm Password (with live match indicator)
Keep me logged in checkbox (default: checked)
On submit, stores all data in sessionStorage as accountData, then routes to the appropriate Step 3 page
STEP 3: Complete Profile (varies by type)

For Locals (/signup/local):

Date of Birth (date picker)
Hometown (Country > State > City smart location picker)
"I'm new to this area" checkbox
Interests (select at least 7 from preset list + custom comma-separated)
Community Pledge checkbox
Submits to POST /api/register with userType: "local"
For Travelers (/signup/traveling):

Date of Birth
Hometown (same smart location picker)
"I'm new to hometown" checkbox
Travel Destination (Country > State > City picker) - unique to travelers
Return Date (must be today or future, year >= 2026)
Interests (at least 7)
Community Pledge
Submits to POST /api/register with userType: "traveler", isCurrentlyTraveling: true
For Businesses (/signup/business):

Business Name, Contact Person Name, Contact Phone
Username & Email (pre-filled from Step 2, disabled)
Business Type (dropdown from preset list + custom option)
Customer Phone, Business Email (optional), Website (optional)
Street Address, Zip Code, City/State/Country (smart location)
GPS Location capture button
Interests & Activities (optional for businesses)
Community Pledge
Submits to POST /api/register with userType: "business"
Login Page (/auth):

Email or Username field
Password field (with show/hide toggle)
"Sign In" button → POST /api/auth/login
"Forgot password?" link → /forgot-password
"Not a Current User? Sign up Here" link → /signup
API Endpoints for Native App:

GET /api/check-username/{username} - returns { available: true/false }
POST /api/register - creates account, returns { user, token }
POST /api/auth/login - login, returns { ok: true, user }
