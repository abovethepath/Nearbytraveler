What was built
Phase 1A — Tonight in [City] widget
New endpoint: GET /api/tonight/:city
Runs 3 parallel queries: events happening today, current available_now count, and travelers with active travel plans. Cached 10 minutes. Returns nothing for empty cities — no fake data.

New component: client/src/components/TonightWidget.tsx
Three sections that each appear only if they have data:

🟢 Open to Meet — green pill showing live count, taps to /quick-meetups
📅 Events Tonight — up to 5 events with time + venue, each links to the event detail page
✈️ Travelers Here Now — avatar strip for up to 6 people, tappable to profiles, "+N more" overflow button
The widget shows itself only when at least one section has data. If tonight is empty everywhere, it hides completely.

Placement: Top of the right sidebar on the home page, above CityArrivalsWidget, above Available Now. It's the first thing you see in the sidebar when something is happening tonight.

Phase 1B — Available Now push notification + activity log
Modified: POST /api/available-now in server/routes.ts

After the response is sent (non-blocking, fire-and-forget):

Writes an activity_log entry (went_available_now) — enriches the feed for future city-feed work
Finds users in the same city who have cityActivityAlerts = true and a valid Expo push token — using either hometown match or active travel plan
Sends a push: "[FirstName] is available to meet in [City] right now." (or anonymous if no first name)
Capped at 50 recipients, errors swallowed per-user so one bad token doesn't stop others
No spam risk — the existing available_now flow already clears prior sessions before creating a new one, so a user can only trigger one push batch per activation.

Phase 2 — People You Almost Met
New endpoint: GET /api/people-almost-met (authenticated)
Single SQL query joining travel_plans on matching city + overlapping dates, left-joining connections to exclude already-connected users. Returns up to 15 results with: name, photo, shared city, overlap label (e.g. "Barcelona in March"), days of overlap, hometown.

New component: client/src/components/PeopleAlmostMet.tsx
Compact card list showing:

Avatar + name
Orange line: "You were both in Barcelona in March"
Hometown (if available)
Connect button (sends connection request inline, no page nav needed)
Shows up to 5 cards with a "See X more" button. Hides completely if no results. Only shown to non-business users.

Placement: Right sidebar, below CityArrivalsWidget.
