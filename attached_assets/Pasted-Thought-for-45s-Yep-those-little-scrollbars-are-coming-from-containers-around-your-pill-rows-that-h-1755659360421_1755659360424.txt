Yep—those little scrollbars are coming from containers around your pill rows that have a height cap (max-h-*) and/or overflow-auto (or a Radix ScrollArea). On mobile that makes each row show a tiny vertical scrollbar. Let’s switch those pill groups to wrap, never scroll.

1) One tiny CSS helper (once)

Add to your global CSS (e.g., globals.css):

.no-scrollbar { -ms-overflow-style: none; scrollbar-width: none; }
.no-scrollbar::-webkit-scrollbar { display: none; }

2) Make pill rows wrap (no more inner scrollbars)

Where you render the pills (e.g., “Countries you’ve both visited”, “Shared languages”, interests, etc.), replace any wrapper like:

<div className="max-h-40 overflow-y-auto ...">  // or ScrollArea, or overflow-auto
  {items.map(/* badges */)}
</div>


with:

<div className="flex flex-wrap items-center gap-2 overflow-visible min-w-0">
  {items.map((txt: string) => (
    <Badge
      key={txt}
      variant="secondary"
      className="inline-flex items-center justify-center h-7 rounded-full px-3 text-xs font-medium
                 whitespace-nowrap"
    >
      {txt}
    </Badge>
  ))}
</div>


Notes

flex flex-wrap → pills wrap to the next line (no need to scroll).

min-w-0 helps truncation inside flex parents.

overflow-visible guarantees no scrollbars in the row.

If you still want a scrolling variant on desktop only, do:

<div className="flex flex-wrap gap-2 sm:hidden">{/* pills */}</div>
<div className="hidden sm:block overflow-x-auto no-scrollbar">
  <div className="flex gap-2">{/* same pills */}</div>
</div>

3) Make sure the parent card/content allows wrapping

On the card that holds those rows:

<Card className="overflow-visible">
  <CardContent className="overflow-visible min-w-0">
    {/* sections with pill rows */}
  </CardContent>
</Card>


Remove any overflow-hidden/overflow-auto on parents of pill rows.

4) Consistent pill size (optional, keeps things tidy)

If some pills look taller/shorter:

<Badge className="inline-flex h-7 items-center rounded-full px-3 text-xs font-medium leading-none">
  {label}
</Badge>


Do those swaps in the sections where you see the scrollbars (likely in your WhatYouHaveInCommon rows and any “pills” lists inside the profile). That will remove the tiny vertical bars and keep everything readable on mobile. If you paste the JSX for one of those pill sections, I’ll hand you the exact patched block for it.
