Those little white up/down “scrollbars” on your pills are the browser’s native form controls (select/number spinners) or a forced scrollbar from overflow:auto on the chip element. Let’s hard-disable both cases everywhere the chips appear.

Below is a single, copy-paste fix. It injects CSS into <head> so it wins over anything else. It also includes tiny usage notes for events/business chips and will not affect unrelated UI.

1) Drop this file in and mount it once

src/GlobalHotfixes.tsx

import { useEffect } from "react";

export default function GlobalHotfixes() {
  useEffect(() => {
    if (document.getElementById("nearby-hotfix")) return;
    const css = `
/* ===== Nearby Hotfix (chips/pills, events, map, wrapping) ===== */

/* --- CHIPS / PILLS: remove scrollbars, native arrows, borders, backgrounds --- */
/* Use the class "chip" on your pill/chip/badge elements (or wrappers). */
.chip {
  overflow: visible !important;                 /* never show a scrollbar */
  background: var(--chip-bg, transparent);      /* customizable if needed */
  border: 0 !important;
  box-shadow: none !important;
  border-radius: 9999px;                         /* nice pill look if you want */
  padding: 0.25rem 0.6rem;
  line-height: 1.1;
  display: inline-flex;
  align-items: center;
  gap: .35rem;
}

/* Kill WebKit scrollbars if something added overflow */
.chip::-webkit-scrollbar { display: none !important; }
.chip { scrollbar-width: none !important; }

/* If a SELECT is used to render a chip, remove native arrow */
.chip select {
  appearance: none !important;
  -webkit-appearance: none !important;
  -moz-appearance: none !important;
  background: transparent !important;
  border: 0 !important;
  padding: 0 !important;
  margin: 0 !important;
}
/* IE/Edge legacy */
.chip select::-ms-expand { display: none !important; }

/* If a NUMBER input sneaks in, remove spinners */
.chip input[type="number"]::-webkit-outer-spin-button,
.chip input[type="number"]::-webkit-inner-spin-button {
  -webkit-appearance: none !important;
  margin: 0 !important;
}
.chip input[type="number"] { -moz-appearance: textfield !important; }

/* Defensive: any element that *looks* like a chip but is positioned absolute */
[class*="chip"][class*="absolute"],
[class*="badge"][class*="absolute"],
[class*="pill"][class*="absolute"] { position: static !important; }

/* --- EVENTS: ensure pills are below the image on all screens --- */
.event-card .pills { position: static !important; inset: auto !important; margin-top: .5rem !important; }
.event-card .image-gradient { display: none !important; } /* avoid text overlap */

/* --- WRAPPING HELPERS (apply as classes where text was spilling) --- */
.wrap-any { overflow-wrap: anywhere !important; word-break: break-word !important; }
.minw0    { min-width: 0 !important; }
.text-left-important { text-align: left !important; }

/* --- MAP: visible container + children fill --- */
.map-shell { position: relative; width: 100%; min-height: 320px; border-radius: .5rem; overflow: hidden; }
@media (min-width: 640px) { .map-shell { min-height: 400px; } }
.map-shell > * { height: 100% !important; width: 100% !important; }

/* --- HERO blue words (scoped) --- */
.hero-clean h1 .hero-blue,
.hero-clean h1 .hero-blue::before,
.hero-clean h1 .hero-blue::after {
  color: rgb(59 130 246) !important; /* blue-500 text only */
  background: transparent !important;
  border: 0 !important;
  box-shadow: none !important;
  padding: 0 !important;
  border-radius: 0 !important;
  content: none !important;
}

/* Kill phantom horizontal scrollbars */
html, body { overflow-x: hidden !important; }
`.trim();

    const style = document.createElement("style");
    style.id = "nearby-hotfix";
    style.textContent = css;
    document.head.appendChild(style);
  }, []);
  return null;
}


Mount it once near the top of your app (e.g., in App.tsx):

import GlobalHotfixes from "@/GlobalHotfixes";

export default function App() {
  return (
    <>
      <GlobalHotfixes />
      {/* ...your routes/layout... */}
    </>
  );
}

2) Use the chip class on your pills (one-line changes)

Wherever you render those “SINGLE”, “United States”, “English”, etc. elements:

{/* BEFORE (example) */}
// <Badge>United States</Badge>  OR  <span className="...">United States</span>

{/* AFTER */}
<span className="chip bg-gray-900/80 text-white">United States</span>
<span className="chip bg-gray-900/80 text-white">Canada</span>
<span className="chip bg-purple-600 text-white">SINGLE</span>
<span className="chip bg-gray-800 text-white">English</span>


If you must keep a <select> for accessibility, wrap it:

<span className="chip bg-gray-900/80 text-white">
  <select value="United States" onChange={()=>{ /* noop */ }}>
    <option>United States</option>
  </select>
</span>


The hotfix strips the native arrow + scrollbars on that select so it looks like a plain pill.

3) Event cards: add two class hooks (no layout rewrite)

Add event-card to your card wrapper.

Put all your tag elements inside a container with class pills.

<article className="event-card ...">
  <div className="relative">
    <img ... />
    {/* your old absolute tag row can stay — hotfix forces it to static */}
    <div className="pills flex flex-wrap gap-2 px-4 pt-3">
      <span className="chip bg-gray-900/80 text-white">{ev.category}</span>
      <span className="chip bg-purple-600 text-white">{ev.attendingCount} attending</span>
      <span className="chip bg-gray-200/20 text-white">Local</span>
    </div>
  </div>
  {/* ...content... */}
</article>


That’s it — the CSS makes those pills stack below the image on every screen so they cannot overlap your text.

4) Business/event text that was overflowing

Wherever text was spilling, add the helpers:

{/* titles/rows that got cut: */}
<div className="minw0 text-left-important">
  <span className="wrap-any">2702 Main St, Austin, Texas, United States</span>
</div>

<p className="wrap-any">
  Cozy neighborhood café serving artisanal coffee and fresh pastries…
</p>

5) Map container

Wrap your map with .map-shell (height guaranteed), and make sure the map root fills 100%:

</div>


Inside InteractiveMap:

return <div id="map-root" style={{ height: "100%", width: "100%" }} />;

Use this markup (the hotfix removes any badge styles):

Why this will fix your screenshot

The white arrows were from native form controls or forced overflow on your chip elements. The .chip rules remove both (appearance + scrollbars), regardless of your component library.

Event tags can’t overlap anymore because the CSS forces them to be static flow content.

Addresses and bios use wrap-any and minw0 so they never push past the viewport.

The map gets a real height via .map-shell.

Do those three edits (mount GlobalHotfixes, add chip class to your pills, add event-card/pills hooks), hard refresh (Cmd/Ctrl-Shift-R), and the up/down boxes will be gone. If one specific component still shows arrows, tell me that component’s JSX name and I’ll paste a targeted patch for it.
